# «Научный клубок» — граф знаний R&D по горно-металлургии

**Хакатон Норникель, задача 2.** Система превращает разнородный корпус R&D-документов
(отчёты, статьи, презентации, таблицы) в верифицированный граф знаний Neo4j и отвечает
на многопараметрические инженерные запросы — с цитатой, источником, датой и уровнем
достоверности у **каждого** факта.

| Метрика | Значение |
|---|---|
| Корпус | **1288 документов** (включая журналы и материалы конференций) |
| Граф | **76 831 узел**, **446 998 связей**, онтология 8/8 типов, 13 типов рёбер |
| Числовых параметров | **53 701** |
| Провенанс | **100%** фактов: цитата + источник + достоверность + дата |
| Точность извлечения (golden set) | precision **0.991** |
| Точность на журналах (3-слойная защита) | **99.95%** — самый шумный источник |
| Отклик графа (окрестность сущности) | **~0.01 с** (денормализация) |
| Тёплый поиск end-to-end | ~1.5–3 с |
| REST API | **42 эндпоинта**, FastAPI + Swagger (`API.md`) |
| Веб-интерфейс | React + Vite + React Flow (отдаётся тем же сервером) |
| Авторизация | JWT + bcrypt, 5 ролей, аудит |

## Почему это не «просто RAG»

Большинство команд подадут LLM-summary поверх векторного поиска. Мы сознательно пошли другим путём:

1. **Детерминированное извлечение, не LLM.** Числовые параметры извлекаются грамматикой +
   газетиром (`src/grammar.py`, `src/gazetteer.py`). Результат воспроизводим бит-в-бит:
   один и тот же корпус → один и тот же граф. Никаких галлюцинированных значений —
   LLM в конвейере только **режет** сомнительное (см. п. 3), никогда не порождает факты.
2. **Провенанс 100%.** Каждый факт в графе несёт дословную цитату, документ-источник,
   confidence и дату извлечения (`src/graph.py:_load_facts`). Любой ответ проверяется
   кликом до исходного предложения.
3. **Трёхслойная защита точности на журналах** (мультистатейные выпуски — главный источник
   мусора у наивных пайплайнов; довели точность до **99.95%**):
   - сегментация выпусков по ГОСТ-якорям УДК (`src/journal.py`);
   - детерминированные гварды: market/DOI/гарбл (`src/guards.py`);
   - LLM-верификатор фактов против их собственных цитат (deepseek, T=0, fail-open, `src/verify.py`).
4. **Таблицы через Vision OCR.** Значимая доля фактов живёт в таблицах, которые plain-text-экстракция
   уничтожает. Мультимодальная модель Qwen3-VL распознаёт составы (`src/vision.py`).
5. **Санкционно-чистый стек.** DeepSeek-V3 (chat) + bge-m3 (эмбеддинги 1024d) +
   Qwen3-VL-235B (vision) — всё open-weight/Китай, через **RouterAI** (routerai.ru),
   работает на территории РФ без санкционных рисков.

## Быстрый старт (Docker — одна команда)

Граф едет готовым дампом Neo4j, переизвлечение не нужно. Подробно — `DEPLOY.md`.

```bash
cp .env.example .env          # впишите ROUTERAI_API_KEY
docker compose up -d --build  # сборка + загрузка дампа графа + старт
# → http://localhost:8000      веб-интерфейс + REST API (вход: admin / admin123)
```

Один контейнер отдаёт и React-интерфейс, и API на порту 8000. Первый вход —
`admin` / значение `ADMIN_PASSWORD` из `.env`; смена пароля в интерфейсе (аватар → «Сменить пароль»).

### Локально без Docker (разработка)
```bash
pip install -r requirements.txt && python -m spacy download ru_core_news_sm
python -m playwright install chromium          # для вкладки «Внешние источники»
uvicorn src.api:app --port 8000                # API + SPA (если собран frontend/dist)
cd frontend && npm install && npm run dev      # dev-фронт на :5173 (HMR)
```

## Возможности интерфейса
- **Поиск** — вопрос обычным языком → факты с числами и точной цитатой из источника; фильтры (гео/материал/процесс/достоверность); экспорт MD / JSON-LD / PDF.
- **Граф знаний** — интерактивная окрестность сущности (React Flow), клик по узлу → детали и число источников, глубина 1–3.
- **Качество** — автоматически найденные противоречия между источниками (Россия vs мир, метод vs метод).
- **Аналитика** — дашборд руководителя: покрытие, география, эксперты, зоны риска.
- **Внешние источники** — поиск статей в CyberLeninka и загрузка их PDF в граф в один клик.
- **Пользователи** — управление учётными записями и ролями (админ).

## Архитектура (кратко)

Подробно — `ARCHITECTURE.md`. Конвейер извлечения:

```
корпус (PDF/DOCX/PPTX/XLSX)
  → extract (текст+страницы)  → normalize (омоглифы/дегифенация/колонтитулы)
  → journal (сегментация выпусков по УДК)         ┐
  → gazetteer (сущности) → grammar (числовые      │ трёхслойная
     параметры) + vision (таблицы, OCR)           │ точность:
  → guards (market/DOI/гарбл — детерминизм)       │ [1] сегментация
  → verify (LLM-судья цитат, T=0, fail-open)      │ [2] гварды
  → contradictions (CONTRADICTS / VALIDATED_BY)   │ [3] LLM-режет
  → graph.load → Neo4j                            ┘
```

**Онтология**: 8 типов узлов ТЗ (Material, Process, Equipment, Property, Experiment,
Publication, Expert, Facility), 13 типов рёбер.

**Поиск — 3 дорожки** (`src/search.py:search`), результаты сливаются:
числовая (интент+числа → Cypher по графу) · семантика (bge-m3, 1024d) ·
keyword-fallback по всему корпусу (`src/kwindex.py`) → merge → фильтры → RBAC (5 ролей) → ответ.

## Соответствие ТЗ (ключевое)

| Требование ТЗ | Где реализовано |
|---|---|
| Многопараметрический поиск (материал+процесс+условия+гео+время) | `src/search.py:search`, `_apply_filters` |
| Онтология 8 типов узлов, связи | `src/graph.py:_label_for`, `_load_edges` |
| Числовые ограничения («сульфаты ≤300 мг/л») | `src/grammar.py`, `src/search.py:_numeric_track` |
| Провенанс фактов (цитата/источник/дата/достоверность) | `src/graph.py:_load_facts` |
| Подсветка противоречий | `src/contradictions.py` → рёбра CONTRADICTS/VALIDATED_BY |
| Пробелы знаний | `src/search.py:_combo_gaps`, `_lang_geo_gaps` |
| Эксперты и лаборатории по теме | `src/search.py:_expert_track` |
| Рекомендации (похожие кейсы, смежные темы) | `src/search.py:build_recommendations` |
| Ручная правка с автором и датой | `src/curation.py` |
| RBAC (5 ролей) + аудит | `src/search.py:_apply_rbac`, `src/obs.py` |
| Подписки/уведомления | `src/notify.py` |

| REST API для фронтенда | `src/api.py` (42 эндпоинта, контракт в `API.md`) |
